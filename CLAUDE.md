# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Stack

Laravel 13 (PHP 8.3+) + Inertia.js + React 19 + TypeScript, styled with Tailwind v4 and shadcn/ui (`radix-sera` style, `olive` base, `hugeicons` icon library — see `components.json`). SQLite by default. Auth is handled by **Laravel Fortify** (login, registration, password reset, email verification, 2FA via `TwoFactorAuthenticatable`). Originated from the `laravel/react-starter-kit`.

## Commands

Development:
- `composer dev` — runs `php artisan serve`, `queue:listen`, `pail` (logs), and `npm run dev` concurrently. This is the standard way to run the app locally.
- `composer setup` — first-time setup (install, key:generate, migrate, npm install, npm run build).
- `php artisan migrate` / `php artisan migrate:fresh --seed` — database is `database/database.sqlite`.

Quality gates (matches CI in `.github/workflows/`):
- `composer ci:check` — runs the full gate: `npm run lint:check`, `npm run types:check`, then `composer test`.
- `composer test` — clears config, runs `pint --test`, then `php artisan test` (Pest).
- `./vendor/bin/pest --filter "test name"` — single test. Pest tests in `tests/Feature` auto-use `RefreshDatabase` (see `tests/Pest.php`).
- `composer lint` (`pint --parallel`) — PHP autofix; `npm run lint` (`biome check --write .`) — JS/TS autofix incl. format + import organize.
- `npm run lint:check` — `biome check .` (lint + format check, no writes). `npm run format` / `format:check` are also available if you only want formatting.
- `npm run types:check` — `tsc --noEmit`.

Build: `npm run build` (or `npm run build:ssr` for SSR).

## Architecture

### Inertia + Wayfinder bridge

Inertia is the glue: Laravel controllers return `Inertia::render('page/name', $props)`, and the React app resolves the name to a page component under `resources/js/pages/`. Layouts are applied centrally in `resources/js/app.tsx` based on the page-name prefix:
- `welcome` → no layout
- `auth/*` → `AuthLayout`
- `settings/*` → `[AppLayout, SettingsLayout]` (nested)
- everything else → `AppLayout`

So a new page named `foo/bar` lives at `resources/js/pages/foo/bar.tsx` and is rendered via `Inertia::render('foo/bar', ...)`. Per-page breadcrumbs are set via `Component.layout = { breadcrumbs: [...] }` (see `pages/dashboard.tsx`).

**Wayfinder generates typed route + controller helpers from PHP into the JS side.** Three generated trees, all gitignored-style and excluded from ESLint/Prettier — **never edit by hand**:
- `resources/js/routes/**` — route URL builders (`import { dashboard } from '@/routes'`).
- `resources/js/actions/**` — typed controller-action wrappers usable with Inertia's `<Form>` (e.g. `ProfileController.update.form()` in `pages/settings/profile.tsx`).
- `resources/js/wayfinder/**` — runtime helpers (query params, URL defaults).

These are regenerated by the `wayfinder` Vite plugin (see `vite.config.ts`) when PHP routes/controllers change. If you add a route/controller, just keep `npm run dev` running.

### Shared Inertia props

`app/Http/Middleware/HandleInertiaRequests.php` injects `auth.user`, `name`, and `sidebarOpen` into every page. The TS shape is declared on `InertiaConfig.sharedPageProps` in `resources/js/types/global.d.ts` — extend that interface when you add new shared props so `usePage().props` stays typed.

### Auth flows (Fortify)

`app/Providers/FortifyServiceProvider.php` wires Fortify's auth views to Inertia pages (`auth/login`, `auth/register`, `auth/two-factor-challenge`, etc.) and registers the `CreateNewUser` / `ResetUserPassword` actions in `app/Actions/Fortify/`. Rate limiters for `login` and `two-factor` are defined here too. Web routes for auth are mostly *not* in `routes/web.php` — Fortify registers them. Only app-level routes (`/`, `dashboard`, settings) are in `routes/web.php` + `routes/settings.php`.

### Backend layout

- `app/Http/Controllers/Settings/{Profile,Security}Controller.php` — settings pages.
- `app/Http/Requests/Settings/*` — FormRequests with validation; reusable rule sets live in `app/Concerns/{Password,Profile}ValidationRules.php`.
- `app/Models/User.php` — uses PHP-attribute `#[Fillable]` / `#[Hidden]` (Laravel 13 style) plus Fortify's `TwoFactorAuthenticatable`.
- Bootstrap config is in `bootstrap/app.php` (middleware stack, Inertia + appearance middleware appended to web group).

### Frontend conventions

- Path alias: `@/*` → `resources/js/*` (`tsconfig.json`).
- shadcn primitives live in `resources/js/components/ui/` and are excluded from lint — treat as vendored.
- `cn()` utility is in `resources/js/lib/utils.ts`; Prettier's Tailwind plugin recognizes `clsx`, `cn`, `cva` (`.prettierrc`).
- React Compiler is enabled (babel plugin in `vite.config.ts`) — avoid manual `useMemo`/`useCallback` micro-optimizations; let the compiler handle them.
- Toasts: server flashes with `Inertia::flash('toast', [...])` and the client surfaces them via `use-flash-toast` + the `<Toaster />` mounted in `app.tsx`.

### Lint quirks worth knowing

JS/TS tooling is **Biome** (`biome.json`) — single tool for format + lint + import sort. It respects `.gitignore` (so the wayfinder-generated `routes/`, `actions/`, `wayfinder/` trees are auto-skipped); `resources/js/components/ui/**` and `resources/views/mail/**` are also explicitly excluded.

Things to know:
- `useImportType` (style) — separate `import type { ... }` statements are required.
- `useBlockStatements` (style) — single-line `if` without braces is rejected.
- `useSortedClasses` (nursery, warn) — Tailwind classes inside `clsx`, `cn`, `cva` are auto-sorted.
- Imports are organized via `assist.actions.source.organizeImports`; runs as part of `biome check`.
- Formatter: 4-space indent, 80 col, single quotes, semicolons. Biome does **not** format YAML or Markdown — those files pass through untouched.
